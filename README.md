##  项目简介

##  项目核心亮点
该项目区别于普通入门级 Demo，主打**工程化优化、场景化落地、问题针对性解决**，核心亮点如下：

### 1. 双层文档净化，解决知识库冗余问题
- 设计 **MD5 精确去重 + 语义相似度去重** 双层清洗机制
- 彻底过滤完全重复文本与语义高度冗余内容，解决金融文档版本多、重复内容杂的行业痛点
- 最终过滤**35%+ 冗余文本**，极大提升知识库纯度与检索效率

### 2. 自研双层检索路由，解决检索不准、召回杂乱问题
- 摒弃传统单一向量检索模式，设计 **关键词精准匹配 + LLM 语义兜底** 双层检索架构
- 结构化问题走关键词精准定位知识库，模糊口语化问题走 LLM 语义理解兜底
- 无关信息召回率从 28% 降至 7%，**问答准确率提升至 91%**

### 3. 场景化知识库隔离，适配金融业务落地
- 按业务维度拆分**信贷、催收、风控独立知识库**，实现场景隔离检索
- 搭配向量库懒加载机制，解决多知识库内存占用过高问题，工程化适配性更强

### 4. 完整多轮对话能力，贴合真实使用场景
- 基于 LangChain 对话记忆组件实现上下文关联问答
- 增加窗口裁剪策略，规避超长对话上下文溢出问题，稳定支持多轮业务咨询

### 5. 工程化重构 & 配置化解耦（本次重点升级）
- 所有业务配置、模型参数统一抽离至 YAML，集中管理
- 提示词模板外置解耦，无需改代码即可调优话术
- 各层逻辑完全解耦，高内聚低耦合，便于后续迭代扩展
- 全覆盖单元测试，保障重构后功能稳定无回归bug

### 6. 接口服务化 & 可直接演示上线
- 基于 FastAPI 封装标准后端接口，结构规范、易对接前端
- 结合 Dify 搭建可视化问答前台，无需前端开发即可快速交付演示
- 前后端解耦架构，支持快速迭代、部署与二次开发

整体采用「**数据预处理 → 知识库构建 → 智能检索 → LLM 生成 → 对话服务 → 测试保障**」标准 RAG 工程链路：
1. **数据层**：批量解析 PDF/TXT/MD 金融业务文档，双层清洗去重
2. **向量层**：基于向量嵌入模型生成文本向量，存入 Chroma 向量库并构建 HNSW 索引加速检索
3. **检索层**：双层路由策略，精准匹配业务场景知识库，过滤无效召回
4. **生成层**：结合检索上下文 + 用户问题，LLM 生成精准、低幻觉领域回答
5. **服务层**：FastAPI 封装接口，Dify 搭建可视化对话应用，完成落地演示
6. **测试层**：37 个单元测试覆盖核心流程，保障迭代稳定性

##  项目难点与解决方案
#### 难点1：金融文档重复冗余、版本杂乱，导致检索干扰大
 解决方案：自研 MD5 精确去重 + 语义相似度去重双层机制，同时清理格式乱码、无效字符，从源头提升知识库质量。

#### 难点2：用户提问口语化、模糊化，单一检索方式效果差
 解决方案：设计双层路由检索策略，结构化问题精准匹配，模糊问题 LLM 语义兜底，兼顾检索速度与精度，大幅降低幻觉。

#### 难点3：多场景知识库并存，内存占用高、加载缓慢
 解决方案：采用场景化知识库隔离 + 向量库懒加载机制，按需加载资源，优化系统性能，贴合企业工程落地思维。

#### 难点4：配置散落在代码、提示词硬编码、难以维护迭代
 解决方案：全量抽离至 YAML 配置，提示词外置解耦，业务逻辑分层，搭配单元测试保障重构稳定性。
